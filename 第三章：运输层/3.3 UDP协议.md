# 3.3 UDP 协议

UDP 优点:

1. 关于发送什么数据以及何时发送的 应用层控制更为精细
2. 无需建立连接
3. 无连接状态
4. 分组首部开销小

缺点: 丢包率高, 过分使用 UDP 会挤压 TCP, 因为无拥塞控制

UDP 报文结构:

```
           32bit
|-----------------------|
    源端口号 目的端口号
    长度     检验和

    应用数据(报文)
```

长度是指 UDP 报文段中的字节数

计算检验和是提供差错检测功能的, 但不能修复

若检验到差错, 有两种实现: 直接丢弃; 送到目的主机并在应用程序中给出警告
